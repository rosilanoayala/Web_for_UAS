📄 HTML Tutorial

💡 Cara Belajar HTML yang Benar

Banyak pemula gagal bukan karena sulit, tetapi karena cara belajar yang salah.

  • ❌ Menghafal tag tanpa memahami fungsi
  • ❌ Tidak langsung praktik
  • ❌ Takut error

Cara yang benar:

  • ✔ Ketik → lihat hasil → ulangi
  • ✔ Pahami struktur, bukan hafalan
  • ✔ Error = bagian dari belajar

HTML itu seperti menyusun Lego. Semakin sering mencoba, semakin cepat paham.

1. HTML Introduction

Sebelum kita mulai membuat website, ada satu pertanyaan penting: “Apa itu HTML?”

HTML (HyperText Markup Language) adalah bahasa dasar yang digunakan untuk menyusun struktur sebuah halaman web.

Artinya, HTML menentukan:

  • Apa yang menjadi judul
  • Apa yang menjadi paragraf
  • Apa yang menjadi gambar

💡 Analogi Sederhana

Agar lebih mudah dipahami, bayangkan sebuah website seperti tubuh manusia:

  • HTML → kerangka (tulang)
  • CSS → tampilan (kulit & warna)
  • JavaScript → perilaku (gerakan)

Tanpa HTML, website tidak memiliki bentuk sama sekali.

📌 Cara Browser Membaca HTML

Browser membaca HTML dari atas ke bawah, baris demi baris. Kemudian browser menyusun struktur yang disebut:

DOM (Document Object Model)

DOM ini bisa dibayangkan seperti pohon:

<body> <h1>Judul</h1> <p>Paragraf</p> </body>

Penjelasan:

  • body → elemen utama (parent)
  • h1 → anak dari body (child)
  • p → anak dari body (child)

Struktur seperti ini disebut hierarki.

💡 Semakin kamu memahami struktur ini, semakin mudah kamu belajar HTML ke tahap berikutnya.

📌 Contoh Sederhana

Mari kita lihat contoh HTML paling dasar:

<h1>Halo Dunia</h1> <p>Ini website pertama saya</p>

Penjelasan per baris:

  • <h1> → digunakan untuk judul besar
  • <p> → digunakan untuk paragraf

Hasil di browser:

Halo Dunia

Ini website pertama saya

🎯 Tujuan Materi Ini

Setelah memahami bagian ini, kamu seharusnya sudah bisa:

  • Memahami apa itu HTML
  • Mengetahui fungsi HTML dalam website
  • Membuat struktur sederhana (judul & paragraf)

🧪 Latihan Kecil

Coba sendiri:

  • Buat judul dengan tulisan "Belajar HTML"
  • Buat paragraf di bawahnya

💡 Tips: jangan takut salah — error adalah bagian dari belajar.

⚠ Kesalahan Umum Pemula

  • Mengira HTML adalah bahasa pemrograman ❌
  • Mengira HTML bisa membuat logika seperti if/loop ❌
  • Hanya membaca tanpa mencoba ❌
  • Tidak memahami hubungan parent dan child ❌

✔ HTML hanya mengatur struktur, bukan logika program.

🚀 Insight Penting

Banyak orang merasa HTML itu mudah, tetapi justru di sinilah fondasi paling penting.

Jika kamu memahami HTML dengan baik:

  • Belajar CSS akan lebih mudah
  • Belajar JavaScript akan lebih cepat
  • Membuat website akan lebih terstruktur

👉 Jadi jangan terburu-buru, pahami dasar ini dengan benar.

2. Struktur Dasar HTML

Sekarang kita masuk ke bagian yang sangat penting: struktur dasar HTML.

Jika HTML diibaratkan sebagai rumah, maka struktur ini adalah pondasi dan rangkanya.

Tanpa struktur yang benar, browser bisa:

  • Menampilkan halaman dengan error ❌
  • Membaca kode secara tidak konsisten ❌

📌 Contoh Struktur Dasar

Berikut adalah bentuk dasar dari semua halaman HTML:

<!DOCTYPE html> <html lang="id"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Judul Halaman</title> </head> <body> Konten halaman </body> </html>

🧠 Cara Memahami Struktur Ini

Jangan langsung menghafal. Pahami dulu urutannya:

  • DOCTYPE → memberi tahu jenis dokumen
  • html → pembungkus seluruh halaman
  • head → informasi tambahan (tidak terlihat)
  • body → isi yang tampil di layar

💡 Urutan ini selalu sama di hampir semua website.

📌 A. DOCTYPE

<!DOCTYPE html> digunakan untuk memberi tahu browser bahwa dokumen ini menggunakan HTML5.

Jika tidak ditulis:

  • Browser bisa masuk ke mode lama (quirks mode)
  • Tampilan bisa berbeda di setiap browser

✔ Jadi DOCTYPE wajib ada di baris pertama.

📌 B. Tag <html>

Ini adalah elemen utama (root) dari seluruh dokumen HTML. Semua elemen lain berada di dalamnya.

Atribut penting:

<html lang="id">

Fungsinya:

  • Memberi tahu bahasa halaman
  • Membantu SEO
  • Membantu screen reader

📌 C. Tag <head>

Bagian ini berisi informasi tentang halaman, tetapi tidak terlihat langsung oleh pengguna.

Contoh isi head:

  • meta charset → agar teks tidak rusak
  • viewport → agar responsive di HP
  • title → judul di tab browser

💡 Anggap head sebagai “pengaturan halaman”.

📌 D. Tag <body>

Semua yang terlihat oleh pengguna berada di sini.

Contoh:

  • Teks
  • Gambar
  • Tabel
  • Form

✔ Jika tidak ada di body, tidak akan terlihat di browser.

🎯 Tujuan Materi Ini

Setelah memahami bagian ini, kamu seharusnya bisa:

  • Mengenali struktur dasar HTML
  • Mengetahui fungsi tiap bagian
  • Membuat halaman HTML sederhana

🧪 Latihan Kecil

Coba buat file HTML sendiri:

  • Tulis struktur dasar HTML
  • Ganti title menjadi namamu
  • Tulis "Halo Dunia" di body

Lalu buka di browser dan lihat hasilnya.

⚠ Kesalahan Umum Pemula

  • Menulis struktur tidak lengkap ❌
  • Lupa menutup tag ❌
  • Menaruh konten di dalam head ❌

✔ Pastikan semua konten berada di dalam body.

🚀 Insight Penting

Struktur HTML adalah fondasi dari semua website.

Jika bagian ini tidak dipahami dengan baik:

  • CSS akan sulit dipelajari
  • JavaScript akan membingungkan

👉 Jadi kuasai bagian ini sebelum lanjut.

❌ Jangan gunakan tabel untuk layout website ✔ Gunakan CSS untuk layout modern

3. HTML Elements

Setelah memahami struktur dasar HTML, sekarang kita masuk ke inti dari HTML: elemen.

Elemen HTML adalah bagian-bagian yang menyusun isi halaman web. Semua yang kamu lihat di website berasal dari elemen HTML.

💡 Apa itu Elemen?

Elemen HTML biasanya terdiri dari 3 bagian:

  • Tag pembuka
  • Isi (content)
  • Tag penutup
<p>Ini paragraf</p>

Penjelasan:

  • <p> → tag pembuka
  • Ini paragraf → isi
  • </p> → tag penutup

✔ Semua bagian ini disebut satu elemen.

📌 Cara Berpikir (Penting)

Jangan hanya menghafal tag. Pahami bahwa setiap elemen memiliki fungsi.

Contoh:

  • <h1> → untuk judul
  • <p> → untuk paragraf
  • <img> → untuk gambar

👉 Jadi HTML itu bukan soal hafalan, tapi soal memahami fungsi.

📌 Contoh Nyata

<h1>Belajar HTML</h1> <p>Saya sedang belajar elemen HTML</p>

Hasil di browser:

Belajar HTML

Saya sedang belajar elemen HTML

📌 Elemen Kosong (Void Elements)

Tidak semua elemen memiliki penutup. Beberapa elemen disebut void elements.

<br> <img src="gambar.jpg"> <hr>

Penjelasan:

  • <br> → membuat baris baru
  • <img> → menampilkan gambar
  • <hr> → garis horizontal

✔ Elemen ini tidak memiliki tag penutup.

📌 Elemen Bersarang (Nested Elements)

Elemen bisa berada di dalam elemen lain. Ini disebut nested (bersarang).

<p> Ini adalah <b>teks tebal</b> </p>

Penjelasan:

  • <b> berada di dalam <p>
  • <p> adalah parent
  • <b> adalah child

💡 Ini penting karena HTML bekerja seperti struktur pohon (DOM).

🎯 Tujuan Materi Ini

Setelah bagian ini, kamu seharusnya bisa:

  • Memahami apa itu elemen HTML
  • Membedakan tag pembuka dan penutup
  • Mengenali elemen kosong
  • Memahami konsep parent dan child

🧪 Latihan Kecil

Coba lakukan ini:

  • Buat 1 judul menggunakan <h1>
  • Buat 1 paragraf menggunakan <p>
  • Tambahkan teks tebal di dalam paragraf

Contoh target:

<p>Saya belajar <b>HTML</b></p>

⚠ Kesalahan Umum Pemula

  • Lupa menutup tag ❌
  • Menulis tag tidak berurutan ❌
  • Menganggap semua elemen punya penutup ❌

✔ Pastikan struktur rapi dan berpasangan.

🚀 Insight Penting

Elemen adalah “batu bata” dari HTML.

Semakin kamu paham elemen:

  • Semakin mudah membaca kode orang lain
  • Semakin cepat belajar CSS & JavaScript

👉 Kuasai elemen = kuasai dasar web development.

4. HTML Attributes

Setelah memahami elemen HTML, sekarang kita masuk ke: attribute.

Jika elemen adalah “bentuk”, maka attribute adalah pengaturan tambahan pada elemen tersebut.

💡 Apa itu Attribute?

Attribute memberikan informasi tambahan pada elemen HTML. Attribute selalu ditulis di dalam tag pembuka.

Struktur umum:

<tag nama="nilai">

Contoh:

<a href="https://google.com">Google</a>

Di sini:

  • href = nama attribute
  • https://google.com = nilai

📌 Cara Berpikir (Penting)

Bayangkan seperti ini:

  • Elemen = benda
  • Attribute = pengaturan benda

Contoh:

  • <img> → gambar
  • src → gambar diambil dari mana

👉 Jadi attribute itu menentukan “bagaimana elemen bekerja”.

📌 Contoh Nyata

<a href="https://google.com" target="_blank"> Google </a>

Penjelasan:

  • href → tujuan link
  • target="_blank" → buka di tab baru

Coba klik:

Google

📌 Attribute yang Sering Digunakan

  • id → identitas unik (1 elemen saja)
  • class → kelompok elemen (bisa banyak)
  • src → sumber file
  • alt → teks alternatif gambar
  • href → link tujuan

💡 Attribute ini akan sering kamu gunakan saat belajar CSS dan JavaScript.

📌 Contoh pada Gambar

<img src="komodo.jpg" alt="Komodo">

Penjelasan:

  • src → lokasi gambar
  • alt → deskripsi jika gambar gagal dimuat

🎯 Tujuan Materi Ini

Setelah bagian ini, kamu seharusnya bisa:

  • Memahami fungsi attribute
  • Menggunakan attribute dasar (href, src, alt)
  • Membedakan id dan class

🧪 Latihan Kecil

Coba lakukan ini:

  • Buat link ke YouTube
  • Buat agar link terbuka di tab baru

Target:

<a href="https://youtube.com" target="_blank"> YouTube </a>

⚠ Kesalahan Umum Pemula

  • Lupa tanda kutip pada nilai ❌
  • Salah menulis nama attribute ❌
  • Mengira semua attribute sama ❌

✔ Attribute harus ditulis dengan format: nama="nilai"

🚀 Insight Penting

Attribute adalah jembatan antara HTML dengan:

  • CSS (melalui class & id)
  • JavaScript (melalui id & atribut lainnya)

👉 Jika kamu paham attribute, belajar CSS dan JavaScript akan jauh lebih mudah.

5. HTML Styles (CSS Dasar)

Setelah memahami HTML, kita masuk ke bagian yang membuat tampilan menjadi menarik: CSS (Cascading Style Sheets).

CSS digunakan untuk mengatur tampilan seperti:

  • Warna teks
  • Ukuran
  • Jarak
  • Posisi

👉 HTML mengatur struktur, CSS mengatur tampilan.

💡 Cara Kerja CSS

CSS bekerja dengan memilih elemen HTML, lalu memberikan gaya.

selector { property: value; }

Contoh:

p { color: blue; font-size: 20px; }

Penjelasan:

  • p → selector (target elemen)
  • color → property
  • blue → value

📌 3 Cara Menggunakan CSS

1. Inline CSS

<p style="color:blue;">Teks</p>

✔ Cepat dan langsung ❌ Tidak disarankan untuk proyek besar

2. Internal CSS

<style> p { color: blue; } </style>

✔ Cocok untuk satu halaman

3. External CSS (RECOMMENDED)

<link rel="stylesheet" href="style.css">

✔ Digunakan untuk banyak halaman ✔ Lebih rapi dan profesional

📌 Contoh Nyata

<p style="color:blue; font-size:20px;"> Teks Biru </p>

Contoh Teks Biru

🎯 Tujuan Materi Ini

  • Memahami fungsi CSS
  • Mengetahui cara kerja selector
  • Membedakan inline, internal, dan external

🧪 Latihan Kecil

Coba ubah:

  • Warna teks
  • Ukuran teks

Gunakan inline style terlebih dahulu.

⚠ Kesalahan Umum

  • Menulis semua style di inline ❌
  • Tidak memahami selector ❌
  • Mencampur HTML dan CSS berlebihan ❌

✔ Gunakan external CSS untuk proyek nyata.

🚀 Insight Penting

CSS adalah langkah berikutnya setelah HTML.

Jika kamu memahami:

  • Selector
  • Property
  • Value

👉 Maka kamu sudah siap masuk ke desain web modern.

6. Headings dan Paragraph

Ketika seseorang membuka sebuah halaman web, hal pertama yang ia cari bukanlah kode, melainkan arah.

Di sinilah heading dan paragraph mulai bekerja—diam, tetapi menentukan bagaimana sebuah informasi dipahami.

📌 A. Heading (h1 – h6)

Heading adalah penanda arah. Ia tidak hanya memperbesar teks, tetapi memberi tahu: “di sinilah kamu berada”.

HTML menyediakan enam tingkat heading. Dari yang paling penting hingga yang paling kecil.

  • <h1> → pusat perhatian
  • <h2> → pembuka bagian
  • <h3> → penjelasan lebih dalam
  • <h4> – <h6> → detail yang semakin spesifik

Sebuah halaman yang baik tidak menggunakan heading secara acak. Ia disusun seperti cerita yang memiliki bab, sub-bab, dan isi.

<h1>Judul Artikel</h1> <h2>Pendahuluan</h2> <h3>Latar Belakang</h3>

Saat browser membaca ini, ia tidak melihat ukuran teks. Ia melihat struktur.

Dan struktur itulah yang membuat halaman bisa dipahami, baik oleh manusia maupun mesin.

❗ Heading bukan alat dekorasi. Ia adalah penunjuk makna.

📌 B. Paragraph (<p>)

Jika heading adalah arah, maka paragraph adalah perjalanan.

Di dalam paragraph, informasi mulai dijelaskan, diperlambat, dan dipahami.

<p>Ini paragraf pertama.</p> <p>Ini paragraf kedua.</p>

Setiap paragraph berdiri sendiri, membawa satu ide utama.

Terlalu panjang, dan pembaca akan lelah. Terlalu pendek, dan makna bisa hilang.

Maka keseimbangan menjadi penting.

  • Satu paragraf → satu ide
  • Gunakan pemisah untuk memberi napas
  • Biarkan mata bergerak dengan nyaman

📌 C. Hubungan Keduanya

Heading dan paragraph tidak bekerja sendiri.

Mereka seperti peta dan jalan.

  • Heading → menunjukkan ke mana harus pergi
  • Paragraph → menjelaskan apa yang ada di sana

Tanpa heading, halaman terasa datar. Tanpa paragraph, halaman terasa kosong.

Keduanya membentuk alur yang membuat pembaca tetap bertahan.

📌 D. Contoh Tampilan

<h1>Judul Besar</h1> <h2>Sub Judul</h2> <p>Isi paragraf</p>

Judul Besar

Sub Judul

Ini contoh paragraf yang menjelaskan isi dari sub judul di atas.

Jika diperhatikan, mata kita secara alami:

  • Melihat judul terlebih dahulu
  • Lalu turun ke penjelasan

Ini bukan kebetulan. Ini adalah cara manusia membaca.

🚀 Catatan yang Sering Terlewat

Banyak pemula menggunakan heading hanya untuk memperbesar teks.

Namun sebenarnya:

  • Ukuran → diatur oleh CSS
  • Heading → menentukan makna

Dan ketika makna diabaikan, struktur pun ikut hilang.

Pada akhirnya, HTML bukan hanya tentang menampilkan sesuatu, tetapi tentang membuat sesuatu dimengerti.

7. HTML Formatting

Teks bukan hanya untuk dibaca. Teks juga bisa dibentuk.

Dalam HTML, proses ini disebut formatting — sebuah cara untuk memberi penekanan, arah, dan rasa pada tulisan.

📌 A. Visual vs Makna

Tidak semua formatting diciptakan sama. Beberapa hanya mengubah tampilan, sementara yang lain membawa arti.

  • <b> → terlihat tebal
  • <strong> → penting
  • <i> → terlihat miring
  • <em> → memberi penekanan

Perbedaannya halus, tetapi penting.

<b> dan <i> berbicara pada mata. <strong> dan <em> berbicara pada makna.

Dan dalam dunia web modern, makna selalu lebih berharga.

📌 B. Membentuk Teks

Berikut beberapa cara HTML “membentuk” tulisan:

<b>Bold</b> <i>Italic</i> <mark>Highlight</mark> <small>Small</small> <del>Deleted</del> <ins>Inserted</ins>

Hasilnya:

Bold
Italic
Highlight
Small
Deleted
Inserted

Setiap elemen membawa nuansa yang berbeda.

📌 C. Karakter dari Setiap Elemen

  • <mark> → menarik perhatian, seperti stabilo
  • <small> → suara kecil, informasi tambahan
  • <del> → sesuatu yang ditinggalkan
  • <ins> → sesuatu yang baru ditambahkan

HTML tidak hanya menampilkan teks, tetapi juga memberi konteks pada perubahan.

📌 D. Cara Berpikir Seorang "Formater"

Seorang penulis biasa menulis kalimat. Seorang formater membentuk pengalaman membaca.

Ia bertanya:

  • Bagian mana yang penting?
  • Bagian mana yang perlu disorot?
  • Bagian mana yang hanya tambahan?

Dari situlah formatting digunakan, bukan sekadar untuk memperindah, tetapi untuk membantu pembaca memahami.

⚠ Catatan Penting

Formatting bukan untuk layout.

Jika ingin:

  • Mengatur posisi → gunakan CSS
  • Mengubah desain → gunakan CSS

HTML tetap fokus pada struktur dan makna.

🚀 Insight

Ketika seseorang membaca halaman web, mereka tidak membaca semua kata.

Mereka memindai. Mereka mencari bagian penting.

Dan di situlah formatting bekerja — diam, tetapi menentukan apa yang terlihat terlebih dahulu.

👉 Formatting yang baik tidak terasa, tetapi selalu membantu.

8. Comments

Ada bagian dalam HTML yang tidak pernah terlihat oleh pengguna, tetapi sering menjadi penyelamat bagi pembuatnya.

Itulah komentar.

📌 Apa itu Komentar?

Komentar adalah catatan di dalam kode yang tidak akan ditampilkan oleh browser.

<!-- Ini komentar -->

Ia tidak mengubah tampilan. Ia tidak mempengaruhi fungsi. Namun ia membantu memahami.

📌 Mengapa Komentar Dibutuhkan?

Ketika kode masih sedikit, semuanya terasa jelas.

Namun seiring waktu:

  • Struktur semakin panjang
  • Elemen semakin banyak
  • Logika semakin kompleks

Dan di titik itu, ingatan mulai tidak cukup.

Komentar hadir sebagai pengingat— bukan untuk komputer, tetapi untuk manusia.

📌 Fungsi Komentar

  • Menjelaskan bagian kode
  • Memberi petunjuk untuk diri sendiri di masa depan
  • Membantu orang lain memahami struktur
  • Menonaktifkan sementara bagian tertentu

📌 Contoh Penggunaan

<!-- Header utama halaman --> <header> ... </header>

Komentar ini mungkin terlihat sederhana, tetapi ia memberi konteks.

Tanpa komentar, kode hanya kumpulan tag. Dengan komentar, kode menjadi cerita.

📌 Menonaktifkan Sementara Kode

Komentar juga bisa digunakan untuk “mematikan” kode tanpa menghapusnya.

<!-- <p>Ini tidak akan tampil</p> -->

Ini berguna saat mencoba sesuatu tanpa merusak struktur utama.

⚠ Hal yang Sering Terlewat

Komentar memang tidak terlihat di halaman, tetapi tetap bisa dilihat melalui:

  • View Page Source
  • Inspect Element

Jadi:

  • Jangan simpan password ❌
  • Jangan simpan data sensitif ❌

🧠 Sebuah Kebiasaan Kecil

Banyak pemula tidak menulis komentar karena merasa “masih ingat”.

Namun beberapa hari kemudian, kode yang sama terasa asing.

Di situlah komentar menjadi penting.

Bukan untuk hari ini, tetapi untuk versi dirimu di masa depan.

🚀 Insight

Kode yang baik bisa dijalankan. Kode yang diberi komentar bisa dipahami.

Dan dalam dunia nyata, dipahami sering kali lebih berharga daripada sekadar berjalan.

10. Bekerja dengan Gambar di HTML

Gambar adalah salah satu elemen paling penting dalam sebuah halaman web. HTML menyediakan beberapa cara canggih untuk mengelola gambar, mulai dari membuat area interaktif pada gambar, menjadikan gambar sebagai latar belakang, hingga menampilkan gambar yang responsif sesuai ukuran layar.

🖼️ 1. Image Map (Peta Gambar Interaktif)

Dengan image map, satu gambar dapat memiliki beberapa area yang dapat diklik, masing-masing mengarah ke tautan yang berbeda. Ini sangat berguna untuk membuat diagram interaktif, peta, atau menu berbasis gambar.

<img src="komodo.jpg" usemap="#map1"> <map name="map1"> <area shape="rect" coords="0,0,100,100" href="https://google.com"> </map>

🔍 Cara Kerja:

  • usemap → menghubungkan gambar dengan peta (map).
  • area → menentukan area yang dapat diklik.
  • shape dan coords → menentukan bentuk dan koordinat area (rect, circle, poly).

⚠️ Koordinat dihitung berdasarkan ukuran asli gambar dalam piksel.

🎮 Demo Interaktif: Atur Tampilan Gambar

Gunakan kontrol di bawah untuk mengubah ukuran, posisi, float, dan sudut gambar. Ini akan membantu Anda memahami bagaimana CSS dapat memengaruhi gambar sebelum membuat image map.





🎨 2. Background Image (Gambar Latar Belakang)

Tidak semua gambar harus menjadi elemen <img>. Terkadang gambar lebih cocok digunakan sebagai latar belakang suatu elemen atau seluruh halaman. CSS menyediakan properti background-image untuk keperluan ini.

body { background-image: url('komodo.jpg'); background-size: cover; background-position: center; background-repeat: no-repeat; }

🔍 Penjelasan Properti:

  • cover → gambar akan memenuhi seluruh area, mungkin terpotong sedikit.
  • contain → gambar akan ditampilkan sepenuhnya tanpa terpotong.
  • center → gambar diposisikan di tengah.
  • no-repeat → gambar tidak diulang-ulang.

🎮 Demo Interaktif: Ubah Latar Halaman

Masukkan URL gambar (lokal atau online) dan klik tombol untuk mengubah latar belakang halaman ini.


💡 Gunakan gambar dari internet atau lokal seperti "komodo.jpg".

📱 3. Picture Element (Gambar Responsif)

Tidak semua layar memiliki ukuran yang sama. Tag <picture> memungkinkan browser memilih gambar yang paling sesuai berdasarkan kondisi tertentu (misalnya lebar layar), sehingga menghemat bandwidth dan mempercepat loading.

<picture> <source media="(max-width: 600px)" srcset="small.jpg"> <source media="(max-width: 1200px)" srcset="medium.jpg"> <img src="large.jpg" alt="Deskripsi Gambar"> </picture>

🔍 Cara Kerja:

  • Browser membaca tag <source> dari atas ke bawah.
  • Jika kondisi media cocok, gambar di srcset akan digunakan.
  • Jika tidak ada yang cocok, browser akan menggunakan tag <img> sebagai fallback.

🚀 Keuntungan:

  • Loading lebih cepat di perangkat mobile.
  • Hemat data pengguna.
  • Tampilan optimal di semua ukuran layar.

💡 Insight: Website modern tidak menampilkan gambar yang sama untuk semua device. Dengan <picture>, Anda dapat menyajikan gambar resolusi rendah untuk layar kecil dan gambar resolusi tinggi untuk layar besar, memberikan pengalaman terbaik bagi semua pengguna.

11. Tables

Tidak semua informasi bisa dijelaskan dengan paragraf. Beberapa hal perlu disusun agar dapat dibandingkan, dilihat pola-nya, dan dipahami dengan cepat.

Di sinilah tabel digunakan.

Tabel bukan sekadar garis dan kotak, tetapi cara untuk menata informasi agar menjadi jelas.

📌 Struktur Dasar

Sebuah tabel terdiri dari beberapa bagian utama. Masing-masing memiliki peran yang berbeda, namun saling melengkapi.

  • <table> → wadah utama
  • <tr> → baris
  • <th> → judul kolom
  • <td> → isi data

Struktur ini terlihat sederhana, namun membentuk sistem yang teratur.

<table border="1"> <tr> <th>Nama</th> <th>Umur</th> </tr> <tr> <td>Budi</td> <td>20</td> </tr> </table>

Hasilnya:

NamaUmur
Budi20

Mata kita secara alami membaca dari kiri ke kanan, dan dari atas ke bawah. Tabel memanfaatkan kebiasaan itu.

📌 Menggabungkan Sel

Tidak semua data harus berdiri sendiri. Terkadang, beberapa bagian perlu digabungkan.

HTML menyediakan dua cara:

  • colspan → menggabungkan kolom
  • rowspan → menggabungkan baris
<td colspan="2">Gabungan Kolom</td> <td rowspan="2">Gabungan Baris</td>

Dengan ini, tabel menjadi lebih fleksibel, tidak lagi kaku.

📌 Tampilan dengan CSS

Dulu, tabel sering menggunakan atribut langsung seperti border. Namun dalam praktik modern, tampilan diatur menggunakan CSS.

table { border-collapse: collapse; } th, td { border: 1px solid black; padding: 8px; }

Dengan CSS, tampilan menjadi lebih rapi, lebih fleksibel, dan lebih mudah diatur.

📌 Cara Berpikir

Tabel bukan untuk semua hal.

Gunakan tabel ketika:

  • Data memiliki hubungan baris dan kolom
  • Perbandingan diperlukan
  • Informasi perlu disusun secara sistematis

Jangan gunakan tabel untuk:

  • Layout halaman
  • Mengatur posisi elemen

Karena itu bukan fungsi utamanya.

🧠 Sebuah Pola

Ketika data mulai terlihat berantakan, tabel menjadi cara untuk merapikannya.

Ia tidak menambah informasi, tetapi membuat informasi lebih mudah dipahami.

Dan terkadang, cara kita menyusun data lebih penting daripada jumlah data itu sendiri.

🚀 Insight

Tabel adalah bentuk paling sederhana dari struktur data.

Dan dari struktur sederhana itu, lahir banyak hal yang lebih kompleks.

Memahami tabel bukan hanya tentang HTML, tetapi tentang bagaimana manusia memahami informasi.

12. HTML Forms (Formulir Interaktif)

Jika sebelumnya HTML hanya menampilkan informasi, di sinilah halaman mulai "hidup". Form adalah jembatan antara pengguna dan sistem, memungkinkan pengguna mengirim data ke server.

📌 Struktur Dasar Form

<form action="/proses" method="post"> <label>Nama:</label> <input type="text" name="nama"> <button type="submit">Kirim</button> </form>
  • action → URL tujuan pengiriman data.
  • method → GET (data tampil di URL) atau POST (data tersembunyi).

📌 Jenis Input Penting

  • text, email, password, number, date
  • radio (pilih satu), checkbox (pilih banyak)
  • select dengan <option> untuk dropdown

📌 Label & Atribut "name"

Label (<label>) meningkatkan aksesibilitas dan membuat form lebih mudah digunakan. Atribut name wajib ada agar data dapat dikenali oleh server.

💡 Ingat! Kesalahan umum: tidak menggunakan <label>, lupa atribut name, atau salah memilih tipe input.

📝 Demo Interaktif: Form Pendataan Siswa

Di bawah ini adalah contoh form lengkap yang menggunakan berbagai jenis input. Isi data lalu klik "Kirim Data" untuk melihat hasilnya dalam tabel.







📊 Data Siswa Terkumpul

Nama NIS Email Kelas JK

🚀 Insight

Sebagian besar fitur penting di internet—login, register, pencarian, upload—dibangun dari form. Memahami form dengan baik adalah langkah besar menuju pengembangan web yang interaktif dan profesional.

13. Semantic HTML

Saat pertama belajar HTML, biasanya kita fokus pada: judul, paragraf, gambar, dan warna.

Namun seiring waktu, muncul pertanyaan: “Bagaimana cara menyusun halaman dengan benar?”

Di sinilah semantic HTML mulai berperan.

📌 Apa itu Semantic HTML?

Semantic HTML adalah penggunaan elemen HTML yang memiliki arti atau fungsi yang jelas.

Artinya, kita tidak hanya membuat tampilan, tetapi juga menjelaskan bagian dari halaman tersebut.

📌 Contoh Elemen

  • <header> → bagian atas (judul / logo)
  • <nav> → menu navigasi
  • <main> → isi utama halaman
  • <section> → bagian konten
  • <article> → konten mandiri (artikel, berita)
  • <footer> → bagian bawah

💡 Nama elemen biasanya sudah menggambarkan fungsinya.

📌 Contoh Sederhana

<header>Website Saya</header> <nav>Home | About</nav> <main> <section> <h2>Judul</h2> <p>Isi konten</p> </section> </main> <footer>Copyright</footer>

Walaupun belum diberi CSS, kita sudah bisa memahami strukturnya.

📌 Kenapa Tidak Pakai <div> Saja?

Sebenarnya, semua ini bisa dibuat dengan <div>.

<div>Header</div> <div>Menu</div>

Namun:

  • <div> tidak memiliki arti khusus
  • semantic HTML memberi makna

👉 Perbedaannya mungkin tidak terlihat, tetapi sangat penting bagi sistem yang membaca halaman.

📌 Mengapa Ini Penting?

  • Membantu mesin pencari (SEO)
  • Membantu screen reader (aksesibilitas)
  • Membuat kode lebih rapi

Meskipun belum terasa di awal, ini akan sangat membantu saat proyek menjadi lebih besar.

🧠 Cara Berpikir Sederhana

Saat membuat halaman, coba tanyakan:

  • “Bagian ini untuk apa?”

Bukan:

  • “Bagaimana tampilannya?”

Dari situ, kamu bisa memilih elemen yang tepat.

⚠ Catatan untuk Pemula

Jika masih merasa bingung, itu wajar.

Semantic HTML biasanya mulai terasa penting saat kamu membuat website yang lebih kompleks.

Untuk sekarang, cukup:

  • mengenali elemen
  • mulai menggunakannya perlahan

🚀 Insight

HTML bukan hanya tentang membuat sesuatu terlihat, tetapi juga tentang membuat sesuatu dimengerti.

Dan semantic HTML adalah langkah awal ke arah itu.

14. Div dan Span

Di antara semua elemen HTML, ada dua yang terlihat sangat sederhana, bahkan sering dianggap “tidak penting”.

Namun justru keduanya digunakan hampir di setiap halaman web.

Elemen tersebut adalah: <div> dan <span>.

📌 Apa Itu <div> dan <span>?

Keduanya adalah elemen “umum” yang tidak memiliki makna khusus, berbeda dengan semantic HTML seperti <header> atau <nav>.

Namun justru karena tidak memiliki makna, mereka menjadi sangat fleksibel.

📌 Perbedaan Utama

  • <div> → elemen block (selalu mulai baris baru)
  • <span> → elemen inline (tetap dalam baris)
<div>Ini div</div> <span>Ini span</span>

Div akan “memisahkan” area, sementara span hanya membungkus bagian kecil dari teks.

📌 Fungsi dalam Praktik

<div> biasanya digunakan sebagai:

  • container (pembungkus)
  • pembagi layout
  • struktur halaman

<span> biasanya digunakan untuk:

  • memberi style pada sebagian teks
  • menandai bagian kecil dalam paragraf
<p> Ini adalah <span style="color:red">teks penting</span> </p>

📌 Contoh Nyata

Sebagian besar website sebenarnya dibangun seperti ini:

<div class="container"> <div class="header">Header</div> <div class="content">Isi</div> </div>

Kemudian CSS digunakan untuk mengatur tampilannya.

Artinya:

HTML → struktur CSS → tampilan

📌 Hubungan dengan CSS

Div dan span hampir selalu digunakan bersama CSS.

Tanpa CSS, keduanya terlihat biasa saja. Dengan CSS, keduanya bisa menjadi apa saja.

Mulai dari:

  • layout website
  • card
  • navbar
  • grid

📌 Kapan Menggunakan?

Gunakan <div> ketika:

  • membuat bagian besar
  • mengatur layout
  • membungkus beberapa elemen

Gunakan <span> ketika:

  • mengubah sebagian kecil teks
  • tidak ingin membuat baris baru

⚠ Kesalahan Umum

  • Menggunakan terlalu banyak div tanpa struktur ❌
  • Tidak menggunakan semantic HTML ❌

Div memang fleksibel, tetapi bukan berarti digunakan untuk semua hal.

🧠 Cara Berpikir

Bayangkan div sebagai “kotak besar”, dan span sebagai “pena penanda”.

Div membentuk struktur, span memberi detail.

🚀 Insight

Sebagian besar layout modern sebenarnya tidak rumit.

Ia hanya kombinasi dari:

  • div untuk struktur
  • CSS untuk tampilan

Kesederhanaan inilah yang membuat web menjadi fleksibel.

Dan dari dua elemen sederhana ini, lahir hampir seluruh tampilan website modern.

15. Accessibility

Website tidak hanya dibuat untuk dilihat, tetapi juga untuk digunakan oleh semua orang.

Termasuk mereka yang:

  • tidak bisa melihat dengan jelas
  • menggunakan screen reader
  • memiliki keterbatasan fisik

Inilah yang disebut accessibility.

📌 Apa Itu Accessibility?

Accessibility adalah cara membuat website agar dapat diakses oleh semua orang, tanpa terkecuali.

HTML sebenarnya sudah menyediakan banyak fitur untuk ini, hanya saja sering diabaikan.

📌 Contoh Penting

  • alt pada gambar
<img src="foto.jpg" alt="Siswa sedang belajar">

Alt membantu screen reader menjelaskan gambar.

  • label pada form
<label>Nama</label> <input type="text">

Label membantu pengguna memahami input.

  • struktur heading

Heading (<h1>–<h6>) membantu navigasi, terutama bagi pengguna screen reader.

📌 Contoh Kasus Nyata

Bayangkan seseorang tidak bisa melihat layar, dan hanya mendengar melalui screen reader.

Tanpa alt:

  • gambar hanya disebut “image”

Dengan alt:

  • gambar dijelaskan secara jelas

Perbedaannya sangat besar.

📌 Hal Sederhana yang Bisa Dilakukan

  • Selalu gunakan alt pada gambar
  • Gunakan label pada form
  • Gunakan heading secara berurutan
  • Gunakan semantic HTML

Hal-hal ini terlihat kecil, tetapi berdampak besar.

⚠ Kesalahan Umum

  • Tidak memberi alt pada gambar ❌
  • Menggunakan div tanpa struktur ❌
  • Melewati urutan heading ❌

Kesalahan ini sering terjadi, terutama pada pemula.

🧠 Cara Berpikir

Saat membuat website, coba tanyakan:

  • “Apakah semua orang bisa menggunakan ini?”

Bukan hanya:

  • “Apakah ini terlihat bagus?”

Dari situ, accessibility mulai terbentuk.

🚀 Insight

HTML yang baik bukan hanya untuk dilihat, tetapi juga untuk dipahami oleh semua orang, termasuk mesin dan alat bantu.

Accessibility mungkin tidak terlihat, tetapi selalu terasa.

Dan di situlah kualitas sebuah website benar-benar diuji.

16. Meta & SEO

Tidak semua yang penting terlihat di halaman.

Beberapa bagian justru bekerja di belakang, mempengaruhi bagaimana sebuah halaman dipahami oleh mesin, bukan manusia.

📌 Meta: Informasi yang Tidak Ditampilkan

<meta name="description" content="Belajar HTML">

Meta bukan untuk pengguna. Meta adalah untuk sistem yang membaca halaman.

Ia tidak muncul di layar, tetapi menentukan bagaimana halaman dikenali.

📌 Cara Mesin “Membaca” Halaman

Browser menampilkan HTML. Mesin pencari menafsirkan HTML.

Perbedaannya halus, tetapi menentukan.

Yang terlihat oleh pengguna:

  • judul
  • teks
  • gambar

Yang dilihat mesin:

  • struktur
  • relasi
  • prioritas

📌 Meta sebagai “Sinyal”

Setiap meta tag adalah sinyal.

Bukan instruksi mutlak, melainkan petunjuk.

Dan seperti semua sistem yang kompleks, petunjuk tidak selalu diikuti— tetapi selalu dipertimbangkan.

📌 Contoh yang Sering Digunakan

<meta name="description" content="Website belajar HTML dasar hingga lanjut"> <meta name="keywords" content="HTML, CSS, belajar coding"> <meta name="author" content="Nama Anda">

Beberapa meta bersifat informatif, beberapa lainnya historis, dan sebagian lagi… hanya relevan dalam konteks tertentu.

📌 SEO Bukan Sekadar Kata Kunci

Banyak yang mengira SEO adalah tentang kata.

Padahal yang lebih penting adalah:

  • struktur yang konsisten
  • hierarki yang jelas
  • makna yang tepat

Di sinilah HTML berperan.

📌 Apa yang Sebenarnya Dinilai?

Bukan hanya isi, tetapi bagaimana isi tersebut disusun.

Sebuah halaman dengan struktur yang baik lebih mudah dipahami, dibandingkan halaman yang hanya penuh konten.

Dan pemahaman adalah dasar dari pencarian.

🧠 Cara Membaca Bagian Ini

Jika terasa membingungkan, itu wajar.

Meta dan SEO jarang terasa di awal, karena hasilnya tidak langsung terlihat.

Namun seiring waktu, pengaruhnya mulai terlihat dari:

  • kemudahan ditemukan
  • kejelasan struktur
  • konsistensi halaman

🚀 Insight

HTML yang baik tidak hanya untuk pengguna, tetapi juga untuk sistem yang membaca di balik layar.

Dan sering kali, yang tidak terlihat itulah yang paling menentukan.
