CSS Tutorial

1. Pengantar CSS (CSS Introduction)

CSS adalah singkatan dari Cascading Style Sheets. Dalam bahasa Indonesia, kurang lebih berarti "Lembar Gaya Berjenjang". CSS adalah bahasa yang digunakan untuk mengatur tampilan dan tata letak halaman web. Tanpa CSS, halaman web hanya akan menampilkan teks polos dan gambar tanpa warna, tanpa pengaturan posisi, dan terlihat sangat membosankan.

Bayangkan Anda membangun sebuah rumah. HTML adalah struktur rumahnya: dinding, atap, pintu, jendela. CSS adalah cat dinding, warna pintu, jenis lantai, dan tata letak furnitur. JavaScript adalah interaksinya: lampu yang bisa dinyalakan, pintu yang terbuka, dan tombol bel yang berbunyi. Ketiganya bekerja sama untuk menciptakan pengalaman yang utuh.

🔍 Mengapa CSS Sangat Penting?

  • Pemisahan struktur dan tampilan → HTML hanya fokus pada konten dan makna (ini heading, ini paragraf, ini gambar), sedangkan CSS fokus pada bagaimana semua itu terlihat. Ini membuat kode lebih rapi dan mudah dipelihara.
  • Konsistensi → Dengan CSS, Anda bisa mendefinisikan satu gaya untuk semua judul di seluruh halaman. Jika ingin mengubah warna semua judul, cukup ubah satu baris CSS, bukan ratusan baris HTML.
  • Fleksibilitas dan responsivitas → CSS memungkinkan halaman web menyesuaikan diri dengan ukuran layar yang berbeda (ponsel, tablet, komputer). Ini disebut desain responsif.
  • Kecepatan loading → File CSS bisa di-cache oleh browser. Setelah diunduh sekali, halaman lain dalam situs yang sama akan menggunakan file yang sama, sehingga lebih cepat.

🧠 Cara Kerja CSS (Analogi Sederhana)

Bayangkan Anda memiliki sekumpulan elemen HTML seperti <h1>, <p>, <button>. CSS adalah seperangkat aturan yang mengatakan: "Setiap <h1> harus berwarna biru, setiap <p> harus berukuran 16px, dan setiap tombol harus memiliki latar belakang hijau." Browser kemudian membaca aturan-aturan ini dan menampilkan elemen sesuai dengan instruksi tersebut.

Prosesnya kurang lebih seperti ini:

Browser membaca file HTML → menemukan elemen <h1> Browser membaca file CSS → menemukan aturan untuk h1 Browser menggabungkan (cascade) semua aturan Browser menampilkan h1 dengan gaya yang sudah ditentukan

🎨 Apa Saja yang Bisa Dilakukan dengan CSS?

CSS dapat mengatur hampir semua aspek visual dari sebuah halaman web. Berikut adalah beberapa kemampuan utamanya:

  • Warna → teks, latar belakang, border, bayangan.
  • Ukuran dan jarak → lebar, tinggi, margin (jarak luar), padding (jarak dalam).
  • Tipografi → jenis font, ukuran, ketebalan, gaya (miring), spasi antar huruf, spasi antar baris.
  • Layout → posisi elemen (kiri, kanan, atas, bawah), tampilan blok atau inline, fleksibel (flexbox), grid.
  • Efek visual → bayangan (box-shadow), sudut melengkung (border-radius), transisi (transition), animasi (animation).
  • Responsivitas → mengubah tampilan berdasarkan ukuran layar menggunakan media query.

📝 Contoh CSS Pertama Anda

Mari kita lihat contoh CSS paling sederhana. Anggap Anda memiliki file HTML dengan sebuah paragraf. Anda ingin paragraf tersebut berwarna biru dan memiliki ukuran font 20px. Berikut adalah kode CSS yang akan melakukannya:

p { color: blue; font-size: 20px; }

Penjelasan:

  • p adalah selector (menunjuk ke elemen <p>).
  • color adalah properti (apa yang ingin diubah).
  • blue adalah nilai (bagaimana tampilannya).
  • Tanda kurung kurawal { } mengelompokkan deklarasi.

🧩 Tiga Cara Menempatkan CSS

Ada tiga cara untuk menerapkan CSS ke halaman HTML. Sebagai pemula, Anda akan sering menggunakan cara pertama dan kedua.

  • External CSS (direkomendasikan) → Tulis CSS di file terpisah (misal style.css) lalu hubungkan dengan <link>. Cara ini paling rapi dan mudah dipelihara.
  • Internal CSS → Tulis CSS di dalam tag <style> di dalam <head>. Cocok untuk halaman yang unik.
  • Inline CSS → Tulis CSS langsung di atribut style elemen HTML. Tidak disarankan untuk proyek besar karena sulit diatur.
/* External CSS (file style.css) */ body { background: #f0f0f0; } /* Internal CSS (di dalam tag style) */ <style> h1 { color: navy; } </style> /* Inline CSS (langsung di elemen) */ <p style="color: red;">Teks merah</p>

🌊 Apa Arti "Cascading" (Berjenjang)?

Kata "Cascading" sangat penting. Ini mengacu pada cara browser menentukan gaya mana yang akan diterapkan ketika ada beberapa aturan yang bersaing. Browser akan mengikuti urutan prioritas:

  1. Gaya dari pengguna (user style) bisa override default browser.
  2. Gaya dari pembuat halaman (author style) lebih kuat dari gaya user.
  3. Inline style memiliki prioritas tertinggi.
  4. Jika spesifisitas sama, aturan yang ditulis terakhir akan menang.

Contoh: Jika Anda menulis dua aturan untuk p, aturan kedua akan menggantikan yang pertama.

p { color: red; } p { color: blue; } /* yang ini yang dipakai */

💡 Tips untuk Pemula

  • Mulai dari yang kecil → Coba ubah warna teks, lalu ukuran font, lalu margin. Jangan langsung belajar semua properti sekaligus.
  • Gunakan alat developer browser → Tekan F12 di Chrome/Firefox, Anda bisa melihat dan mengedit CSS secara langsung, serta melihat kotak model (box model) elemen.
  • Jangan takut bereksperimen → CSS tidak akan merusak komputer. Salah satu keindahan CSS adalah bahwa properti yang salah hanya diabaikan, tidak menyebabkan error fatal.
  • Pelajari konsep "box model" sejak awal → Ini adalah kunci untuk memahami layout.

🚀 Kesimpulan

CSS adalah bahasa yang sangat kuat namun tetap bersahabat untuk dipelajari. Setelah Anda menguasai dasar-dasar seperti selector, properti warna, font, margin, dan padding, Anda sudah bisa membuat halaman web yang menarik. Selanjutnya, Anda akan belajar tentang selector yang lebih spesifik, cara mengatur tata letak, dan efek-efek keren. Ingatlah bahwa semua desainer web profesional pun mulai dari langkah pertama yang sama seperti Anda sekarang.

Di bagian selanjutnya, kita akan membahas syntax dasar CSS secara lebih detail. Pastikan Anda sudah paham konsep di atas sebelum melanjutkan.

2. CSS Syntax (Aturan Penulisan CSS)

Syntax adalah aturan penulisan kode. Dalam CSS, setiap aturan terdiri dari dua bagian utama: selector (pemilih) dan declaration block (blok deklarasi). Memahami syntax adalah langkah pertama yang sangat penting karena jika syntax salah, CSS tidak akan bekerja.

🔍 Struktur Dasar CSS

Mari kita lihat contoh paling sederhana:

h1 { color: blue; font-size: 20px; }

Mari kita bedah baris per baris:

  • h1 → Selector. Bagian ini memilih elemen HTML mana yang akan diubah. Di sini, semua elemen <h1> di halaman akan terkena aturan ini.
  • { } → Kurung kurawal membuka dan menutup declaration block. Semua deklarasi (aturan) ditulis di antara kurung ini.
  • color: blue; → Sebuah deklarasi. Terdiri dari property (color) dan value (blue), dipisah oleh titik dua :. Setiap deklarasi diakhiri dengan titik koma ;.
  • font-size: 20px; → Deklarasi kedua. Bisa ada banyak deklarasi dalam satu blok.

📝 Analogi Sederhana

Bayangkan selector adalah "orang yang menerima perintah", dan deklarasi adalah "perintah itu sendiri". Contoh: Jika Anda seorang manajer, Anda memberi instruksi: "Semua karyawan (selector) harus memakai seragam biru (deklarasi)." Dalam CSS, kita bilang: "Semua h1 harus berwarna biru."

🧩 Komponen dalam Satu Deklarasi

  • Property → Nama fitur yang ingin diubah. Contoh: color (warna teks), background-color (warna latar), margin (jarak luar), font-family (jenis huruf). Ada puluhan properti CSS.
  • Value → Nilai yang diberikan untuk properti tersebut. Contoh: blue, #ff0000, 20px, solid. Setiap properti memiliki nilai yang diperbolehkan.
  • Titik koma (;) → Pemisah antar deklarasi. Sangat penting! Jika lupa, dua deklarasi akan dianggap satu dan menyebabkan error.
/* Benar */ p { color: red; font-size: 16px; } /* Salah (lupa titik koma) */ p { color: red font-size: 16px; }

🎯 Aturan Penting dalam CSS Syntax

  • CSS tidak error keras → Jika browser menemukan deklarasi yang salah (misal property tidak dikenal), browser akan mengabaikannya dan melanjutkan ke deklarasi berikutnya. Ini berbeda dengan bahasa pemrograman lain yang langsung berhenti. Karena itu, kadang CSS tidak bekerja seperti yang diharapkan tanpa pesan error jelas.
  • Spasi dan baris baru tidak penting → CSS mengabaikan spasi berlebih dan baris baru. Anda bisa menulis semua dalam satu baris atau banyak baris, asalkan syntax-nya benar. Namun menulis dengan rapi membantu manusia membacanya.
  • Case-sensitive → CSS peka terhadap huruf besar/kecil pada selector dan property? Sebagian besar tidak, tetapi nilai tertentu (seperti URL) peka. Sebagai praktik baik, selalu gunakan huruf kecil.
  • Komentar → Gunakan /* ... */ untuk menulis catatan yang tidak dieksekusi. Komentar sangat membantu saat bekerja dalam tim atau mengingat kode lama.
/* Ini adalah komentar, tidak mempengaruhi tampilan */ body { background: white; /* komentar di samping deklarasi */ }

⚡ Urutan Deklarasi: "Last Rule Wins"

Jika Anda menulis dua deklarasi dengan properti yang sama untuk selector yang sama, maka deklarasi yang terakhir akan digunakan. Ini disebut "last rule wins". Contoh:

h1 { color: red; color: blue; /* yang ini yang dipakai */ }

Hasilnya: semua h1 berwarna biru. Ini penting karena Anda bisa menimpa aturan sebelumnya tanpa menghapusnya.

🧪 Contoh Kesalahan Syntax yang Sering Terjadi pada Pemula

  • Lupa titik koma → Dua deklarasi akan menyatu, browser bingung.
  • Lupa kurung kurawal penutup } → Semua CSS setelahnya akan error.
  • Salah menulis property → Misal colour: blue; (bahasa Inggris Inggris) tidak akan dikenali, diabaikan.
  • Lupa titik dua : antara property dan value → Tidak valid.
/* Kode dengan beberapa kesalahan */ h1 { color: red /* lupa titik koma dan titik dua? Tidak, di sini lupa titik koma */ font-size: 20px; /* akan dianggap bagian dari deklarasi sebelumnya */ } /* Browser akan mengabaikan aturan ini atau salah interpretasi */

💡 Tips Praktis untuk Pemula

  • Gunakan editor kode yang baik → Editor seperti VS Code akan memberi warna berbeda pada selector, property, dan value, serta menandai syntax error.
  • Selalu akhiri deklarasi dengan titik koma → Meskipun deklarasi terakhir dalam blok bisa tanpa titik koma, biasakan selalu pakai untuk menghindari error saat menambah deklarasi baru.
  • Gunakan indentasi → Menulis property dalam baris baru dengan indentasi (spasi atau tab) membuat kode mudah dibaca.
  • Jangan takut bereksperimen → Buka alat developer browser (F12), pilih tab "Elements", dan coba ubah CSS secara langsung. Anda akan melihat efeknya real-time tanpa perlu menyimpan file.

🔗 Contoh CSS dengan Banyak Deklarasi

Berikut adalah contoh CSS yang lebih lengkap untuk memformat sebuah kartu (card):

.card { background-color: white; border: 1px solid #ddd; border-radius: 8px; padding: 20px; margin: 10px; box-shadow: 0 2px 5px rgba(0,0,0,0.1); font-family: 'Arial', sans-serif; }

Setiap baris adalah deklarasi yang mengubah satu aspek tampilan. Perhatikan setiap baris diakhiri titik koma.

📖 Kesimpulan

Syntax CSS sebenarnya sangat sederhana: selector { property: value; }. Setelah Anda memahami pola ini, Anda sudah bisa membaca dan menulis CSS dasar. Tantangan sebenarnya adalah menghafal properti dan nilai yang tersedia, serta memahami bagaimana selector bekerja. Di bagian selanjutnya, kita akan mempelajari berbagai macam selector agar Anda bisa menargetkan elemen mana pun di halaman dengan presisi.

Sebelum lanjut, pastikan Anda sudah mencoba menulis sendiri beberapa aturan CSS sederhana di editor atau alat developer browser. Praktek langsung adalah cara terbaik untuk memahami syntax.

3. CSS Selectors (Pemilih Elemen)

Selector adalah jantung dari CSS. Selector menentukan elemen HTML mana yang akan dikenai gaya. Tanpa selector yang tepat, CSS tidak akan tahu harus mengubah apa. Bayangkan selector seperti alamat rumah – Anda harus memberi tahu kurir ke mana paket harus diantar.

CSS menyediakan banyak jenis selector, mulai dari yang paling sederhana hingga yang sangat kompleks. Sebagai pemula, Anda cukup menguasai 4 jenis selector dasar terlebih dahulu: universal, element, class, dan ID. Setelah itu, Anda bisa belajar selector lanjutan seperti atribut, pseudo-class, dan kombinasi.

🎯 1. Universal Selector (*)

Universal selector ditulis dengan tanda bintang *. Selector ini memilih semua elemen di halaman. Sangat berguna untuk menerapkan gaya global, seperti mengatur ulang margin dan padding bawaan browser.

/* Mengatur semua elemen agar tidak memiliki margin dan padding bawaan */ * { margin: 0; padding: 0; box-sizing: border-box; }

Kapan menggunakannya? Biasanya digunakan di awal file CSS untuk "reset" gaya default browser. Namun hati-hati, karena selector ini memiliki prioritas terendah (bisa ditimpa oleh selector lain).

🏷️ 2. Element Selector (Tag Selector)

Element selector memilih semua elemen berdasarkan nama tag HTML (seperti p, h1, div, a). Ini adalah selector yang paling mudah dipahami.

/* Semua paragraf akan berwarna abu-abu */ p { color: gray; } /* Semua heading level 1 akan berukuran 28px */ h1 { font-size: 28px; }

Kelemahan: Selector ini akan memengaruhi semua elemen dengan tag tersebut. Jika Anda ingin hanya sebagian paragraf yang berbeda, Anda perlu selector yang lebih spesifik (class).

📦 3. Class Selector (.nama-class)

Class selector adalah selector yang paling sering digunakan dalam pengembangan web modern. Class selector ditulis dengan titik . diikuti nama class. Di HTML, Anda memberikan atribut class="..." pada elemen.

/* CSS */ .highlight { background-color: yellow; font-weight: bold; } /* HTML */ <p class="highlight">Teks ini berlatar kuning dan tebal.</p> <p>Teks ini normal.</p>

Keunggulan: Satu class bisa digunakan berkali-kali di berbagai elemen. Anda juga bisa memberikan beberapa class sekaligus, dipisah spasi: class="highlight large".

🆔 4. ID Selector (#nama-id)

ID selector ditulis dengan tanda pagar # diikuti nama ID. Setiap ID harus unik dalam satu halaman (tidak boleh ada dua elemen dengan ID yang sama). ID biasanya digunakan untuk elemen yang benar-benar spesial, seperti header utama, footer, atau bagian yang hanya muncul sekali.

/* CSS */ #main-header { background: navy; color: white; padding: 20px; } /* HTML */ <div id="main-header">Ini adalah header utama</div>

Perbedaan ID dan Class: ID lebih spesifik daripada Class (prioritas lebih tinggi), tetapi karena keunikannya, ID kurang fleksibel. Sebaiknya gunakan class kecuali Anda yakin elemen tersebut hanya ada satu.

⚖️ Prioritas Selector (Specificity)

Pernahkah Anda mengalami kebingungan ketika CSS Anda tidak berfungsi meskipun sudah menulis aturan? Kemungkinan besar itu masalah spesifisitas. Browser memberikan bobot (weight) pada setiap selector. Semakin spesifik selector, semakin tinggi prioritasnya. Urutan prioritas dari yang tertinggi ke terendah:

  1. Inline style (ditulis langsung di elemen) – bobot tertinggi.
  2. ID selector (#id) – sangat spesifik.
  3. Class, attribute, pseudo-class (.class, [type="text"], :hover) – cukup spesifik.
  4. Element selector (div, p) – rendah.
  5. Universal selector (*) – paling rendah.
/* Contoh konflik */ p { color: black; } /* element selector */ .teks { color: blue; } /* class selector (menang atas element) */ #judul { color: red; } /* ID selector (menang atas class) */

Jika dua selector memiliki tingkat spesifisitas yang sama, maka aturan yang ditulis terakhir akan menang (last rule wins).

🔗 Selector Lanjutan (Combinators & Pseudo-class)

Setelah Anda menguasai selector dasar, Anda bisa belajar selector yang lebih powerful untuk menargetkan elemen secara presisi:

  • Descendant combinator (spasi) → div p memilih semua p yang berada di dalam div, di level mana pun.
  • Child combinator (>) → div > p hanya memilih p yang merupakan anak langsung dari div.
  • Adjacent sibling (+) → h1 + p memilih p yang langsung mengikuti h1.
  • General sibling (~) → h1 ~ p memilih semua p yang berada setelah h1 dan memiliki parent yang sama.
  • Pseudo-class (:hover, :focus, :first-child) → menargetkan elemen dalam keadaan khusus.
  • Attribute selector → [type="text"] memilih input dengan tipe text.
/* Contoh selector lanjutan */ div p { color: green; } /* semua paragraf dalam div */ div > p { font-weight: bold; } /* paragraf anak langsung */ a:hover { text-decoration: none; } /* link saat di-hover */ input:focus { border: 2px solid blue; } /* input saat aktif */

💡 Tips Memilih Selector yang Tepat

  • Gunakan class sebanyak mungkin – class fleksibel dan mudah di-maintain.
  • Hindari selector yang terlalu spesifik seperti div#container .box .title. Semakin panjang, semakin sulit di-override.
  • Jangan gunakan ID untuk styling jika memungkinkan, karena ID memiliki prioritas tinggi dan susah diubah nanti. ID lebih cocok untuk JavaScript.
  • Gunakan selector universal (*) hanya untuk reset global, jangan untuk styling elemen spesifik.
  • Pelajari pseudo-class seperti :hover dan :nth-child() untuk membuat interaksi keren.

🧪 Latihan Sederhana

Coba buat file HTML sederhana dengan beberapa paragraf, beberapa div, dan beberapa elemen dengan class. Kemudian buat CSS yang menggunakan berbagai selector di atas. Amati perubahannya. Gunakan alat developer browser (F12) untuk melihat selector mana yang diterapkan.

🚀 Kesimpulan

Selector adalah alat untuk "menunjuk" elemen HTML. Anda telah mempelajari selector universal, element, class, ID, serta sedikit tentang prioritas dan selector lanjutan. Dengan menguasai selector, Anda bisa mengendalikan tampilan halaman web dengan presisi. Di bagian selanjutnya, kita akan menerapkan pengetahuan ini ke properti-properti CSS seperti warna, border, margin, padding, dan box model.

4. CSS Colors (Warna)

Warna adalah elemen desain yang paling mendasar. Dengan CSS, Anda bisa mewarnai teks, latar belakang, border, dan banyak lagi. CSS menyediakan beberapa cara untuk mendefinisikan warna, masing-masing dengan kelebihan tersendiri.

🎨 Metode Warna dalam CSS

  • Named Colors — Ada 140 warna standar seperti red, blue, tomato, darkcyan. Ini sangat mudah diingat dan cocok untuk prototyping. Namun pilihannya terbatas.
  • HEX (Hexadecimal) — Format #RRGGBB, di mana RR=merah, GG=hijau, BB=biru, masing-masing dari 00 sampai FF. Contoh: #ff0000 = merah, #00ff00 = hijau. Ini yang paling populer di kalangan desainer web.
  • RGB / RGBA — rgb(255,0,0) merah, nilai 0-255. rgba(255,0,0,0.5) menambahkan opacity (alpha) antara 0 (transparan) dan 1 (solid). Berguna untuk efek transparan.
  • HSL / HSLA — hsl(0, 100%, 50%) juga merah. HSL lebih intuitif karena Anda memutar roda warna (hue 0-360), lalu mengatur saturasi (0-100%) dan kecerahan (0-100%). Sangat baik untuk membuat palet warna yang harmonis.
/* Contoh semua format */ color: red; /* named */ color: #ff0000; /* hex */ color: rgb(255, 0, 0); /* rgb */ color: rgba(255, 0, 0, 0.5);/* rgba dengan transparansi */ color: hsl(0, 100%, 50%); /* hsl */

🔍 Penjelasan contoh di atas: Setiap baris menunjukkan cara berbeda untuk menghasilkan warna merah. Browser akan membaca format yang Anda tulis dan menampilkannya sama. Perbedaan utama terletak pada fleksibilitas: rgba() memungkinkan transparansi, hsl() memudahkan variasi warna dengan mengubah nilai Hue. Cobalah mengganti angka pada rgb() atau hsl() untuk melihat perubahan warna secara langsung di demo interaktif.

Untuk aksesibilitas, pastikan kontras antara teks dan latar belakang cukup tinggi (gunakan alat online seperti WebAIM Contrast Checker).

5. CSS Cascading & Prioritas

CSS singkatan dari Cascading Style Sheets. Kata "cascading" mengacu pada mekanisme bagaimana browser menentukan style mana yang diterapkan ketika ada beberapa aturan yang menargetkan elemen yang sama. Prioritas dihitung berdasarkan:

  1. Inline style (dalam atribut style) → prioritas tertinggi.
  2. ID selector → #header (sangat spesifik).
  3. Class, attribute, pseudo-class → .box, [type="text"], :hover.
  4. Element selector → div, p, h1.
  5. Universal selector → * (prioritas terendah).

Selain spesifisitas, urutan penulisan juga penting: jika dua selector memiliki spesifisitas yang sama, maka yang ditulis terakhir akan menang (last rule wins).

/* Contoh konflik */ p { color: red; } /* element selector, spesifisitas rendah */ .teks { color: blue; } /* class selector → menang karena lebih spesifik */ #paragraf { color: green; } /* ID selector → menang lebih tinggi */

🔍 Penjelasan contoh konflik: Misalkan ada elemen <p class="teks" id="paragraf">Halo</p>. Ketiga aturan di atas bersaing. Browser menghitung spesifisitas: ID selector (nilai 100) > class selector (nilai 10) > element selector (nilai 1). Maka warna teks akan hijau karena ID menang. Jika ID tidak ada, maka class yang menang (biru). Jika tidak ada class juga, barulah element selector (merah) yang berlaku. Ini adalah fondasi penting untuk memahami mengapa kadang CSS Anda tidak berfungsi seperti yang diharapkan.

💡 Best Practice: Hindari inline style karena sulit di-override. Gunakan class sebanyak mungkin. Jangan terlalu spesifik (misal div#container .box) agar mudah diubah nanti. Gunakan !important hanya sebagai jalan terakhir, karena bisa membuat debugging sangat sulit.

Contoh nyata: Saat Anda ingin mengubah warna semua teks dalam suatu bagian, cukup buat class baru dan letakkan di akhir file CSS, maka akan menimpa aturan sebelumnya.

6. Border (Garis Tepi)

Border adalah garis yang membungkus elemen. Setiap border memiliki tiga properti: ketebalan (width), gaya (style), dan warna (color). Border dapat diterapkan ke semua sisi atau masing-masing sisi (top, right, bottom, left).

/* Shorthand (satu baris) */ border: 2px solid black; /* Terpisah */ border-width: 2px; border-style: solid; border-color: black; /* Per sisi */ border-top: 1px dashed red;

🔍 Penjelasan: Baris pertama (border: 2px solid black;) adalah cara ringkas menggabungkan ketiga properti sekaligus. Urutan bebas, tetapi biasanya: ketebalan, gaya, warna. Baris kedua sampai keempat menunjukkan cara terpisah jika ingin mengubah hanya satu aspek. Baris kelima menunjukkan border hanya di sisi atas dengan gaya putus-putus (dashed) berwarna merah. Anda bisa mencampur gaya yang berbeda untuk setiap sisi.

Jenis-jenis border-style:

  • solid → garis padat.
  • dashed → garis putus-putus (seperti strip).
  • dotted → titik-titik.
  • double → garis ganda.
  • groove, ridge, inset, outset → efek 3D (tergantung browser).

Border Radius (sudut melengkung)

Properti border-radius membuat sudut elemen menjadi tumpul atau lingkaran. Nilai 50% menghasilkan lingkaran sempurna (jika lebar = tinggi). Ini sangat populer untuk membuat tombol bulat, kartu, avatar profil, dan elemen UI modern.

Contoh: border-radius: 10px; akan membuat sudut melengkung 10 piksel. border-radius: 50%; pada elemen persegi akan menjadi lingkaran.

7. Margin (Jarak Luar)

Margin adalah ruang di luar border yang memisahkan elemen dengan elemen lain. Margin tidak memiliki warna dan bersifat transparan. Anda bisa membayangkan margin sebagai "bantalan" yang mendorong elemen lain menjauh.

margin: 20px; /* semua sisi */ margin: 10px 20px; /* atas-bawah 10px, kiri-kanan 20px */ margin: 10px 20px 10px 20px; /* atas, kanan, bawah, kiri */ margin-top: 10px; margin-right: 15px; margin-bottom: 10px; margin-left: 15px;

🔍 Penjelasan contoh kode: Baris pertama memberikan jarak 20px di keempat sisi. Baris kedua memberikan jarak vertikal (atas-bawah) 10px dan horizontal (kiri-kanan) 20px. Baris ketiga adalah cara eksplisit untuk setiap sisi (urutan: atas, kanan, bawah, kiri – searah jarum jam). Baris keempat hingga ketujuh menunjukkan cara mengatur margin hanya untuk sisi tertentu. Margin sering digunakan untuk memberi jarak antar paragraf, antara gambar dengan teks, antar kartu dalam grid, atau untuk memusatkan elemen (margin: auto).

8. Padding (Jarak Dalam)

Padding adalah ruang di dalam border, antara konten dan border. Padding membuat konten tidak menempel pada tepi, meningkatkan kenyamanan baca dan estetika. Berbeda dengan margin, padding mempengaruhi ukuran total elemen (karena menambah lebar/tinggi).

padding: 20px; /* semua sisi */ padding: 10px 20px; /* atas-bawah 10px, kiri-kanan 20px */ padding: 10px 20px 10px 20px;/* atas, kanan, bawah, kiri */ padding-top: 10px;

🔍 Penjelasan contoh kode: Sama seperti margin, padding juga bisa ditulis dalam bentuk shorthand. Perbedaan utama: padding mempengaruhi ukuran total elemen. Contoh: tombol dengan padding: 10px 20px akan terlihat lebih besar dan nyaman diklik. Kartu dengan padding membuat teks tidak menempel ke tepi, sehingga lebih rapi.

9. CSS Box Model (Model Kotak)

Konsep paling fundamental dalam CSS layout adalah Box Model. Setiap elemen HTML sebenarnya adalah sebuah kotak yang terdiri dari 4 lapisan (dari dalam ke luar):

  • Content → isi utama (teks, gambar, dll).
  • Padding → ruang transparan di sekitar konten.
  • Border → garis yang mengelilingi padding.
  • Margin → ruang transparan di luar border.
/* Urutan dari dalam ke luar */ Content → Padding → Border → Margin

🔍 Mengapa box model penting? Karena lebar total elemen = width + padding kiri/kanan + border kiri/kanan + margin kiri/kanan. Banyak pemula mengira width: 200px berarti kotak selebar 200px, tetapi jika diberi padding 20px dan border 5px, lebar sebenarnya menjadi 200 + 40 + 10 = 250px. Ini sering menyebabkan layout rusak.

/* Mengaktifkan border-box untuk semua elemen (sangat direkomendasikan) */ * { box-sizing: border-box; }

Dengan border-box, lebar yang Anda tentukan sudah termasuk padding dan border. Jadi width: 200px; padding: 20px; border: 5px; akan tetap 200px (padding dan border diambil dari dalam). 10. Text Styling (Gaya Teks)

CSS menyediakan banyak properti untuk mengatur tampilan teks, seperti perataan, dekorasi, spasi, dan transformasi huruf. Teks yang baik sangat penting untuk keterbacaan dan pengalaman pengguna.

text-align: left | center | right | justify; /* perataan horizontal */ text-decoration: underline | overline | line-through | none; text-transform: uppercase | lowercase | capitalize; letter-spacing: 2px; /* jarak antar huruf (kerning) */ word-spacing: 4px; /* jarak antar kata */ line-height: 1.5; /* jarak antar baris (leading) */ text-indent: 20px; /* indentasi baris pertama */

🔍 Penjelasan properti:

  • text-align mengatur perataan horizontal teks. justify membuat teks rata kiri-kanan (seperti koran).
  • text-decoration menambahkan garis bawah, garis atas, atau coretan. Biasanya underline untuk link.
  • text-transform mengubah huruf menjadi kapital semua, kecil semua, atau huruf pertama kapital.
  • letter-spacing menambah jarak antar huruf (kerning). Berguna untuk efek judul.
  • word-spacing menambah jarak antar kata.
  • line-height mengatur jarak antar baris. Nilai tanpa satuan (misal 1.5) relatif terhadap ukuran font.
  • text-indent memberikan indentasi pada baris pertama paragraf.

11. Font (Tipografi)

Font menentukan tampilan huruf. CSS memungkinkan kita mengatur jenis font, ukuran, ketebalan, dan gaya. Tipografi yang baik membuat konten lebih mudah dibaca dan menarik.

font-family: 'Arial', sans-serif; /* nama font, fallback */ font-size: 16px; /* ukuran (px, em, rem, %) */ font-weight: normal | bold | 100-900; font-style: normal | italic; font-variant: small-caps; /* huruf kapital kecil */

🔍 Penjelasan properti font:

  • font-family menentukan jenis huruf. Browser akan mencoba font pertama, jika tidak tersedia maka menggunakan font cadangan (sans-serif, serif, monospace). Contoh: 'Arial', sans-serif berarti "pakai Arial, kalau tidak ada pakai sans-serif default".
  • font-size mengatur ukuran huruf. Satuan px (piksel) adalah ukuran tetap. em relatif terhadap ukuran font parent. rem relatif terhadap ukuran font root (html). Rekomendasi: gunakan rem untuk aksesibilitas, karena user bisa mengubah ukuran font default browser.
  • font-weight mengatur ketebalan. Nilai bold atau angka 700 sama. normal atau 400. Angka 100–900 (kelipatan 100).
  • font-style untuk teks miring (italic) atau normal.
  • font-variant: small-caps mengubah huruf kecil menjadi kapital kecil (ukuran lebih kecil dari kapital biasa).

12. Display (Tata Letak Dasar)

Properti display menentukan bagaimana elemen ditampilkan dalam alur dokumen. Nilai yang paling umum:

  • block → elemen mengambil lebar penuh, dimulai dari baris baru. Contoh: div, p, h1. Anda bisa mengatur lebar, tinggi, margin, padding.
  • inline → elemen hanya selebar konten, berada dalam baris yang sama. Contoh: span, a, strong. Tidak bisa diatur lebar/tinggi (diabaikan).
  • inline-block → seperti inline tetapi bisa diatur lebar/tinggi. Sangat berguna untuk membuat menu navigasi horizontal atau grid sederhana.
  • none → elemen tidak ditampilkan (seperti dihapus dari halaman). Ruang yang ditempati juga hilang.
  • flex dan grid → model layout modern yang sangat powerful (dibahas lebih lanjut di tingkat lanjut).
display: block; display: inline; display: inline-block; display: none;

🔍 Penjelasan perbedaan: Elemen block selalu memulai baris baru dan memenuhi lebar container. Elemen inline berada dalam baris yang sama dengan teks di sekitarnya, dan ukuran lebar/tingginya tidak bisa diubah (hanya dipengaruhi oleh konten). inline-block menggabungkan keduanya: ia berada dalam baris (seperti inline) tetapi bisa diatur lebar/tinggi (seperti block).

13. Position (Posisi Elemen)

Properti position mengatur metode penempatan elemen, memungkinkan Anda mengubah posisi elemen dari alur normal. Nilai-nilainya:

  • static (default) → mengikuti alur normal dokumen. Properti top/left/right/bottom tidak berpengaruh.
  • relative → elemen bergeser dari posisi normalnya (menggunakan top, left, dll), tetapi ruang aslinya tetap dipertahankan. Berguna untuk menggeser sedikit atau sebagai anchor untuk absolute.
  • absolute → elemen diposisikan relatif terhadap ancestor terdekat yang memiliki position selain static (biasanya relative). Elemen dihapus dari alur normal, sehingga elemen lain mengisi tempatnya. Sering untuk tooltip, modal, dropdown.
  • fixed → elemen tetap pada posisi relatif terhadap viewport (layar), tidak ikut scroll. Cocok untuk header sticky, tombol back-to-top, atau iklan mengambang.
  • sticky → hybrid antara relative dan fixed. Elemen berperilaku seperti relative hingga mencapai batas scroll tertentu, lalu menjadi fixed. Sangat baik untuk header tabel atau sidebar yang tetap terlihat saat scroll.

Pastikan parent memiliki position: relative agar absolute terikat di dalamnya. Position fixed hati-hati karena bisa menutupi konten jika tidak diberi margin/padding pada body. Gunakan position: sticky untuk membuat navbar atau sidebar yang tetap terlihat saat scroll.

14. Z-Index (Tumpukan Elemen)

Ketika elemen menggunakan position selain static, mereka dapat saling tumpang tindih. Properti z-index menentukan urutan tumpukan (stacking order). Semakin besar nilai, semakin di atas. Nilai default adalah auto (0). Z-index hanya berlaku untuk elemen yang memiliki position: relative/absolute/fixed/sticky.

.atas { position: absolute; z-index: 10; } .bawah { position: absolute; z-index: 5; }

🔍 Penjelasan: Jika dua elemen overlapping, browser akan menampilkan elemen dengan z-index lebih besar di atas. Jika nilai sama, maka elemen yang muncul belakangan di HTML akan berada di atas.

15. Overflow (Mengatasi Konten Berlebih)

Ketika konten di dalam suatu elemen terlalu besar dan melebihi ukuran elemen tersebut, properti overflow menentukan apa yang harus dilakukan. Bayangkan Anda memiliki kotak dengan ukuran tetap, tetapi teks di dalamnya lebih panjang – apakah teks dipotong, muncul scrollbar, atau keluar dari kotak?

overflow: visible; /* default: konten keluar dari kotak */ overflow: hidden; /* konten terpotong, tidak ada scroll */ overflow: scroll; /* selalu tampilkan scrollbar */ overflow: auto; /* scrollbar muncul jika diperlukan */

🔍 Penjelasan: Gunakan overflow: auto untuk membuat scrollbar hanya saat dibutuhkan. overflow: hidden berguna untuk memotong gambar yang terlalu besar. overflow-x dan overflow-y mengatur sumbu horizontal dan vertikal secara terpisah.

16. Float & Clear (Mengatur Elemen Mengambang)

Properti float membuat elemen "mengambang" ke kiri atau kanan, sehingga teks atau elemen lain bisa mengelilinginya. Ini dulu sering digunakan untuk layout kolom, tetapi sekarang lebih disarankan menggunakan Flexbox atau Grid. Namun float masih berguna untuk efek teks membungkus gambar.

float: left; /* mengambang ke kiri */ float: right; /* mengambang ke kanan */ float: none; /* default */ clear: both; /* menghentikan efek float di sekitarnya */

🔍 Penjelasan: Jika Anda memiliki gambar dan ingin teks mengelilinginya, gunakan float: left pada gambar. Masalah umum: elemen parent tidak membungkus anak yang float (collapse). Solusinya gunakan clearfix atau overflow: auto pada parent.

17. Inline-Block (Elemen Sejajar dengan Lebar Teratur)

display: inline-block menggabungkan kelebihan inline (bisa sejajar horizontal) dan block (bisa diatur lebar/tinggi). Sangat berguna untuk membuat menu navigasi horizontal, tombol sejajar, atau kartu dalam baris.

display: inline-block; width: 150px; height: 100px;

🔍 Penjelasan: Elemen inline-block tidak memulai baris baru (seperti inline), tetapi Anda bisa mengatur lebar, tinggi, margin, dan padding (seperti block).

18. Align (Mengatur Posisi Elemen)

Mengatur elemen agar berada di tengah (kiri-kanan atau atas-bawah) adalah kebutuhan umum. CSS menyediakan berbagai cara, tergantung jenis elemen (block, inline, atau flex).

🔹 Horizontal Align (Rata Tengah Horizontal)

  • Elemen block dengan lebar tetap: margin: 0 auto;
  • Teks/inline: text-align: center; pada parent
  • Flexbox: display: flex; justify-content: center;

🔹 Vertical Align (Rata Tengah Vertikal)

  • Menggunakan line-height: untuk teks satu baris, set line-height sama dengan tinggi elemen.
  • Flexbox: display: flex; align-items: center; (metode paling mudah)
  • Position + transform: untuk kasus khusus.
/* Horizontal center untuk block */ .block { width: 200px; margin: 0 auto; } /* Flexbox center (horizontal & vertikal) */ .container { display: flex; justify-content: center; align-items: center; height: 200px; }

19. Combinators (Hubungan Antar Selector)

Combinator digunakan untuk menargetkan elemen berdasarkan hubungannya dengan elemen lain. Ada empat jenis:

  • div p → descendant selector (semua p di dalam div)
  • div > p → child selector (p yang langsung anak div)
  • h1 + p → adjacent sibling (p yang langsung setelah h1)
  • h1 ~ p → general sibling (semua p setelah h1 dengan parent sama)
/* Contoh */ article p { color: blue; } /* semua paragraf dalam article */ article > p { font-weight: bold; } /* paragraf anak langsung */ h2 + p { margin-top: 0; } /* paragraf setelah h2 */

20. Pseudo-classes (Kondisi Khusus Elemen)

Pseudo-class adalah kata kunci yang ditambahkan ke selector untuk menargetkan elemen dalam keadaan tertentu, misalnya saat di-hover, fokus, atau sebagai anak pertama/terakhir.

a:hover { color: red; } /* saat mouse di atas link */ input:focus { border: 2px solid blue; } /* saat input aktif */ li:first-child { font-weight: bold; } /* item pertama dalam list */ li:nth-child(odd) { background: #f0f0f0; } /* item ganjil */

💡 Tips: Pseudo-class sangat berguna untuk interaksi dan styling dinamis tanpa JavaScript. Contoh populer: :hover untuk tombol, :nth-child(even) untuk zebra stripe pada tabel.

21. Pseudo-elements (Gaya pada Bagian Tertentu)

Pseudo-element digunakan untuk menata bagian spesifik dari suatu elemen, seperti baris pertama, huruf pertama, atau menambahkan konten sebelum/sesudah elemen.

p::first-line { font-weight: bold; } /* baris pertama paragraf */ p::first-letter { font-size: 2em; } /* huruf pertama */ .box::before { content: "👉 "; } /* tambah konten sebelum */ .box::after { content: " 👈"; } /* tambah konten sesudah */

22. Opacity (Transparansi Gambar & Elemen)

Properti opacity mengatur tingkat transparansi suatu elemen, dari 0 (sepenuhnya transparan) hingga 1 (sepenuhnya terlihat). Sangat berguna untuk efek hover pada gambar, latar belakang, atau membuat elemen "samar".

img { opacity: 0.5; } /* setengah transparan */ img:hover { opacity: 1; } /* menjadi jelas saat hover */ .box { opacity: 0.3; }

🖌️ Demo Interaktif: Opacity

🧪 Geser slider untuk mengatur transparansi gambar secara manual. Nilai 0 = tidak terlihat, 1 = sepenuhnya terlihat.

🐭 Coba hover ke gambar di atas! Gambar akan menjadi lebih transparan saat kursor di atasnya (efek hover).

💡 Tips: Gunakan efek opacity pada gambar saat hover untuk interaksi yang halus. Perhatikan bahwa opacity mempengaruhi seluruh elemen (termasuk teks di dalamnya). Jika hanya ingin latar transparan, gunakan rgba() atau hsla().
