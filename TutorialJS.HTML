1. Apa itu JavaScript? (Analogi Sederhana)

Bayangkan Anda sedang merakit sebuah Robot Halaman Web:

  • HTML adalah batangnya (Besi, kerangka, tulang, dan posisi tubuh).
  • CSS adalah cangkangnya (Cat warna, kosmetik, baju, dan penampilan luar).
  • JavaScript (JS) adalah Otak dan Ototnya!

Tanpa JavaScript, robot web Anda hanya bisa diam mematung (halaman statis). Dengan JavaScript, robot tersebut bisa membalas sapaan Anda, bergerak, berhitung, mengingat nama Anda, hingga menangani transaksi transaksi kompleks.

💡 Intinya: JavaScript bertugas membuat website Anda menjadi dinamis, interaktif, bisa diklik, memproses data, dan bisa mengambil keputusan secara logis.

Buktikan Sendiri: Sentuh tombol di bawah ini. JavaScript akan langsung merespons sentuhan Anda!

2. Bagaimana Cara Memasang Otak (JS) ke Badan (HTML)?

Sama persis seperti cara HTML menghubungkan CSS menggunakan tag <style>, JavaScript menggunakan tag khusus yaitu <script>. Terdapat 3 cara untuk menyuntikkan JS ke HTML Anda:

  • 1. Internal JS (Tanam Dalam)
    Otak ditanam langsung di dalam badan HTML. Biasanya diletakkan di paling bawah baris HTML, tepat sebelum penutup tag </body>.
  • 2. External JS (Terpisah di Folder)
    Otak disimpan rapi di file terpisah (contoh: logika.js) lalu dihubungkan menggunakan "kabel": <script src="logika.js">. (Ini cara paling disarankan agar kode HTML Anda tidak sesak dan berantakan!).
  • 3. Inline JS (Diselipkan Langsung pada Tombol)
    Memberi perintah singkat langsung di dalam atribut elemen tertentu. (Contoh: pada tombol di atas kita memanggil atribut onclick="...").
<!-- Contoh 1: Internal JS (Menyatu dalam html) --> <script> console.log("Halo, saya otak internal yang bekerja!"); </script> <!-- Contoh 2: External JS (Panggil File Terpisah) --> <script src="logika.js"></script>

🤔 Tanya: Kenapa tag <script> sebaiknya ditaruh di paling bawah HTML (sebelum tag </body> tertutup)?

Jawab: Karena kita harus membiarkan Browser membangun Badannya (kerangka HTML) terlebih dahulu 100%, baru setelah badannya siap, kita hidupkan Otaknya (JS). Jika otak dihidupkan lebih dulu padahal tombolnya belum jadi, JavaScript akan kebingungan (error) karena target benda yang ingin ia rasuki dan ubah tidak 100% jadi!

3. Output (Bagaimana JavaScript Berbicara ke Dunia Luar)

Sebuah otak/program harus bisa berkomunikasi untuk menginfokan hasil hitungannya kepada manusia. JavaScript memiliki 3 cara sakti untuk "berbicara":

  • innerHTML → Cara halus. JS membisikkan kata-kata lalu mencetak/mengubah elemen langsung di dalam halaman HTML pengguna dan merubah halamannya.
  • alert() → Cara teriak. Memunculkan kotak *popup* di tengah layar yang memaksa pengguna menekan tombol "OK" sebelum bisa lanjut mengontrol web.
  • console.log() → Cara rahasia (Khusus Developer). Pengguna biasa tidak akan melihat tulisan ini. Teks ini digunakan oleh Web Developer untuk melacak kesalahan/error. Anda hanya bisa melihatnya bila menekan tombol F12 pada keyboard lalu memilih menu Console pada pojok kanan bawah browser.

4. Variabel: Kardus Penyimpanan Data

Bayangkan Anda sedang pindah rumah. Anda butuh kardus penyimpanan untuk mengamankan barang-barang kecil (nama barang, nilai rapot, umur) sebelum diletakkan di tempatnya. Dalam JavaScript, "kardus virtual" untuk menyimpan nilai data tersebut dinamakan Variabel.

Dalam bahasa modern, hanya ada 2 kardus yang diakui hukum dan 1 kardus usang:

  • let → Kardus fleksibel! Isinya boleh dikeluarkan dan diganti secara bebas sesuai keinginan Anda selama program berjalan. (Gunakan ini paling sering).
  • const → Brankas baja permanen (Konstanta). Sekali Anda menaruh barang ke dalamnya, TIDAK BOLEH DIGANTI ULANG selamanya! (Gunakan ini untuk data absolut/tetap seperti nilai PI 3.14 atau NIK Paspor).
  • var → Kardus lapuk keluaran tahun 90-an sebelum tahun 2015. (Jangan gunakan lagi), karena ia mudah bocor yang menciptakan kerumitan *bug* pada pemrograman saat ini.
// Membuat kardus bernama 'umur' menggunakan let let umur = 16; umur = 17; // ✅ Diizinkan! Isinya diganti. // Membuat brankas bernama 'nama' menggunakan const const nama = "Agus Budiman"; // nama = "Budi"; // ❌ PERINGATAN ERROR: Web akan macet karena Anda melanggar kesucian kardus const!

5. Tipe Data (Jenis-jenis Barang di dalam Kardus)

Jika kardus (variabel) merangkap sebagai tempat simpan, lalu apa jenis barang yang di dalamnya? Berikut adalah 5 barang paling sakral dalam JS:

  • String (Teks) → Dibungkus dengan tanda kutip ganda atau tunggal. Contoh: "Halo Dunia", 'Budi'.
  • Number (Angka) → Angka bebas matematika (tanpa tanda kutip!). Bisa bulat atau desimal. Contoh: 100, 3.14.
  • Boolean (Saklar Lampu) → Hanya punya dua tombol: Hidup atau Mati. Contoh: true (Benar), false (Salah).
  • Array (Rak Sepatu) → Kardus besar bersaf-saf yang bisa menyimpan banyak barang sekaligus yang berurutan. Contoh: ["Apel", "Mangga", "Jeruk"].
  • Object (Kamus/Buku Data) → Barang ajaib yang menyimpan kamus lengkap berisi nama label beserta valuenya. Contoh: {nama: "Andi", umur: 20}.

6. Operator (Tanda Matematika Cerdas)

JS tentu saja sangat pintar berhitung. Berikut adalah logo/simbol ajaibnya:

  • Aritmatika (Hitung): + (Tambah), - (Kurang), * (Kali), / (Bagi), % (Sisa Bagi/Modulus).
  • Perbandingan (Bertanya): == (Apakah Sama?), > (Lebih Besar?), < (Lebih Kecil?), != (Tidak Sama?). Membandingkan dua barang dan menjawab dengan Boolean (True/False).
  • Logika (Syarat): && (DAN: Semua harus benar), || (ATAU: Minimal salah satu benar).

7. Percabangan (If - Else) / Mengambil Keputusan

If (Jika) / Else (Kalau Tidak) adalah cara JS "berpikir dan mengambil keputusan cerdas" murni seperti otak detektif. Ia mengecek kondisi, dan baru bertindak jika syarat terpenuhi.

Analogi: "JIKA (IF) hari ini hujan, maka JS akan membuka payung. KALAU TIDAK HUJAN (ELSE), maka JS memakai kacamata hitam."

let nilai = 80; // JIKA nilai lebih besar atau sama dengan 75 if (nilai >= 75) { console.log("Selamat, Anda Lulus!"); } // KALAU TIDAK else { console.log("Maaf, silakan coba lagi."); }

8. Perulangan (Loop) / Mesin Fotokopi JS

Pernahkah Anda dihukum menulis "Saya berjanji tidak akan nakal" sebanyak 100 kali? Manusia akan lelah, tapi komputer sangat suka pekerjaan berulang! Itulah fungsi Loop.

Gunakan for jika Anda tahu pasti jumlah ulangannya (Misal: Ulang 5 kali). Gunakan while jika Anda tidak tahu kapan harus berhenti sampai sebuah syarat terpenuhi (Misal: Ulang terus TAPI berhenti *while/saat* tombol ditekan).

// Mulai dari 1; Selama kurang dari sama dengan 5; Tambah 1 terus for (let i = 1; i <= 5; i++) { console.log("Cetak angka ke-" + i); }

9. Fungsi (Function) / Resep Masakan

Fungsi ibarat sebuah mesin pembuat Jus atau sebuah Resep Masakan. Anda merakit resepnya sekali saja, dan Anda bisa memanggilnya/mencetak jus tersebut berkali-kali kapanpun Anda mau hanya dengan menyebut namanya!

Analogi: Anda membuat fungsi buatJus(buah). Jika Anda masukkan "Apel", keluar Jus Apel. Masukkan "Mangga", keluar Jus Mangga.

// 1. Membuat Resep (Mendefinisikan Fungsi) function sapaPengunjung(nama) { return "Halo selamat datang, Tuan " + nama + "!"; } // 2. Memanggil Resep let pesan = sapaPengunjung("Andi"); // Hasilnya: Halo selamat datang, Tuan Andi!

10. DOM (Document Object Model) / Kekuatan Telekinesis

Ini adalah ilmu sihir sesungguhnya dari JS! DOM Manipulation adalah kemampuan JS untuk mengendalikan, merubah, menyembunyikan, atau mewarnai elemen HTML secara Real-Time (Langsung di layar) tanpa harus memuat ulang (refresh) halaman!

  • Membidik Sasaran: document.getElementById("idSasarannya") → Mengincar tag HTML dengan ID tersebut.
  • Merubah Teksnya: .innerHTML = "teks baru"
  • Merubah Warnanya: .style.color = "red"

Saya adalah Teks HTML biasa yang tidak berdaya.

11. Event Handler / Detektor Sensor

Web yang diam saja akan membosankan. Event Handler adalah "Sensor" yang kita tempelkan ke elemen HTML. Jika sensor itu mendeteksi aktivitas pengguna (seperti di-klik, disentuh, atau digeser), maka JS akan bereaksi secara instan.

  • onclick: Sensor Ditekan (Diklik).
  • onmouseover & onmouseout: Sensor saat kursor melayang di atas elemen dan saat kursor pergi.
  • onkeyup: Sensor saat tombol keyboard diketik lepas (sangat berguna untuk fitur Search/Cari).

12. Array dan Object (Rak Buku & Lemari Brankas)

Jika Variabel biasa hanya bisa menampung 1 barang, lalu bagaimana jika kita punya ratusan data murid? Gunakan kardus raksasa!

  • Array [ ] → Rak Buku bersaf. Menyimpan banyak barang secara berjejer. Posisi mengambilnya dimulai dari laci nomor 0. (Laci ke-0, Laci ke-1, dst).
  • Object { } → Lemari Berlabel. Menyimpan barang dengan menempelkan nama/stiker (Key) ke barangnya (Value). (Contoh: Laci nama, Laci umur).
// ARRAY: Urutan Berjejer let buah = ["Apel", "Mangga", "Jeruk"]; console.log(buah[0]); // Mengeluarkan mangkuk ke-0 ("Apel") // OBJECT: Menggunakan Label/Nama Ciri let siswa = { nama: "Budi", umur: 17, kelas: "XII" }; console.log(siswa.nama); // Mengeluarkan data berlabel 'nama' ("Budi")

13. LocalStorage / Memori Ingatan Browser

Biasanya, jika Anda menekan F5/Refresh halaman web, semua data JS akan terlupa (Amnesia). LocalStorage adalah memori otak buatan di dalam Google Chrome atau aplikasi HP Anda yang menolak untuk lupa!

Dengan LocalStorage, web bisa mengingat nama Anda atau mode terang/gelap *(Dark Mode)* meskipun komputer sudah dimatikan berhari-hari.

14. Kesalahan Remeh & Tips Bertahan Hidup (Survival)

JavaScript terkadang menyebalkan karena salah ketik koma , saja bisa membuat website mati total tanpa ampun. Berikut pantangan utamanya:

⚠️ Awas:

  • Lupa mendeklarasikan kardus. Langsung menulis umur = 20 tanpa tulisan let atau const di depannya.
  • Sistem *Case-Sensitive* yang kejam. Penulisan GetElement akan dianggap error jika yang benar adalah getElement (huruf G harus kecil!).
  • Menggunakan 1 sama dengan = (memasukkan barang) yang seharusnya === (membandingkan kesamaan).

💡 Tips Dewa JS: Pahlawan terkuat Anda adalah console.log(). Selalu cek pesan error berwarna merah di Menu Console (F12). Karena browser selalu memberitahu baris ke berapa yang salah ketik secara jujur!
